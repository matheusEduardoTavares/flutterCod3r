O Inherited Widget é um tipo de Widget que ajuda a trafegar informação dentro da árvore de componentes da aplicação. O Inherited Widget tem o método .of(context) que é uma das formas que ele tem para acessar dados através de uma árvore de componentes.

Quando temos uma aplicação componentizada temos uma grande árvore de componentes. Chega uma hora que temos alguns dados em um componente e que precisamos passar esses dados para um outro componente mas que pode estar muito distante. Da forma tradicional, temos a comunicação direta ou indireta, ou seja, teria que passar via construtor componente por componente, um por um, ou algum parâmetro (comunicação direta) ou alguma função (callback, comunicação indireta) até ser possível chegar onde queremos tal informação. Isso é inviável de ser feito se a árvore é grande e ou os componentes estão muito distantes. Uma das soluções para isso é usar um gerenciamento de estado, e um desses gerenciamentos é o Provider que se baseia no princípio do Inherited Widget, em que colocamos o Inherited Widget na árvore de componentes e a partir dele conseguimos ter uma comunicação direta a partir de qualquer componente. É possível acessar o Inherited Widget diretamente de algumas formas, e uma dessas formas é usar o método .of(context).

Sabemos que os componentes tem um BuildContext e recebe ele na hora de construir os componentes. Acontece que esses contextos tem relações, são metadados e conseguem se comunicar entre si, então temos uma árvore de componentes e uma árvore de contextos. Como visível temos a árvore de componentes, como invisível temos a árvore de contextos. Se pegarmos um componente por exemplo e usarmos:
MeuWidget.of(context).data , estamos acessando o atributo data desse widget, e ele irá conseguir achar quem é o widget certo devido a árvore de contextos pois vai navegando com ela, de forma a conseguir chegar no Inherited Widget, pega uma instância desse tipo e devolve uma instância de MeuWidget. Internamente ele consegue resolver uma instância do elemento para assim termos acesso aos atributos que precisamos para acessar na aplicação. Esse é um princípio no qual o Provider vai se basear, ele se baseia no widget do tipo Inherited Widget. Nem sempre que usamos o método .of(context) estamos tratando com um Inherited Widget, mas na maioria das vezes é ele quem está sendo usado por debaixo dos panos.
